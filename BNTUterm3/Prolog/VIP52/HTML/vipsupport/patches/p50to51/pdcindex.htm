How to Download the Visual Prolog 5.1 Patch

If you are a registered user of Visual Prolog 5.0, you can follow these instructions to update to Visual Prolog 5.1 version (build 565).
The update is free for all registered users - just install the new version on the top of the old registered version.

Patch Description

The main purpose of Visual Prolog 5.1  is to fix various  bugs reported from the 5.0  and 5.02 versions, but it does also include new  features.

The full version of  Visual Prolog 5.1 is  available only  as  a new Visual Prolog 5.1 CD that you can order from our sales department. 

However, we provide the possibility to update the following directories via patch:

BIN
ESTA
INCLUDE
LIB
DOC_TOOL
VPI\LIB
VPI\INCLUDE
SOCKBIND
SQLBIND
ODBCBIND

Note, please, that the patch does not upgrade examples from the following directories:

DOC_TOOL
SOCKBIND
SQLBIND
ODBCBIND

Please, read Upgrade notes

Installation Instructions

To download the patch:

  1. Visual Prolog 5.1 patch should be applied to the base version of Visual Prolog 5.0 only.
    If you have already applied patches to the versions 5.01 and 5.02, you need to restore Visual Prolog 5.0 from your distribution CD. We recommend you to make maximal installation.
  2. From the Download table  choose:
What version are you going to upgrade (1st Step or Professional)?
What way of downloading to use: via your browser or by FTP?
From what location to download a patch file via your browser: Europe or USA?
  1. Click on  the filename in the table.
  2. The download program will ask you for a destination directory. You should install to the same directory as the registered version. Please, specify the root Visual Prolog directory as a target. Check that this directory has no read-only attribute.
  3. Run a downloaded file that is a self-extracting executable. It will unpack itself and delete work files.
  4. Remove all the directories that are not relevant to you if you wish.
  5. Please, read the Remarks if :
you have changed some source files of the previous version (e.g. examples);
you have chosen an installation different from maximal while Visual Prolog 5.0 installation.
  1. Please, read "Frequently Asked Questions about Downloading Patches" if you have problems in downloading and applying a patch.

Remarks

  1. The patch would not be applied to the files that have been changed by a user (e. g. examples).
    In the list "Results of Application of Patch File" the following message will appear: Old Files Invalid...
    Workaround: If you want to upgrade these files, please, restore them from the Visual Prolog CD, apply a patch, and then insert your changes if necessary.
  2. If while Visual Prolog 5.0 installation, you have chosen an installation different from maximal, then:
  1. The file Patch.ERR would be produced even if there were no errors.
  2. In the list "Results of Application of Patch File" the following message can appear: Old Files Invalid..
    Do not pay attention to this message if you have not changed the source files of the previous version (see item 1 of Remarks).
  3. A patch program would create new directories to locate files that have been added in the last version .
    You can delete unnecessary directories after downloading.
  4. In some cases it takes a long time and consumes a large amount of memory to skip uninstalled files. The system can be hanged-up.
    The workaround is to do a full installation of the version 5.0, apply  the patches, and  then to remove all the directories that are not relevant to you.
