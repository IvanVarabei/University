Caution

Be careful, while upgrading you Visual Prolog 5.1 projects into  Visual Prolog 5.2 projects, if you are going to return later to Visual Prolog 5.1.

Please, use copies of your projects created by previous versions of Visual Prolog, or do not update scripts with the help of the "Warning!" dialog, if you want to test Visual Prolog 5.2 only and then to return to Visual Prolog 5.1.
Do not forget to save your  build scripts if you are regenerating scripts with the help of the "Warning!" dialog in the Visual Prolog 5.2 VDE.

Do not install the Personal Edition of Visual Prolog 5.2 into your current Visual Prolog directory if you use the Professional Edition of Visual Prolog.

Since Visual Prolog 5.1 does not provide support for directories, which names include spaces, we do not recommend you to install Visual Prolog 5.2 into such directories (e.g., "Program Files") or create Visual Prolog 5.2 projects in such directories, in the case, you are going to use these projects under Visual Prolog 5.1.
