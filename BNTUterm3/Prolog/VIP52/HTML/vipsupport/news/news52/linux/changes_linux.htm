Changes from Visual Prolog 5.1 to Visual Prolog 5.2

Linux Version of Visual Prolog

Creating CGI and other Console Applications under the Linux Version of Visual Prolog 5.2

We found out that a lot of Visual Prolog users prefer to build their applications under Linux. That is why, while creating Visual Prolog 5.2, we paid special attention to the Linux version. 
We have fixed several serious bugs, and made changes in the Compiler. We hope that this version will be much more reliable.
The main purpose of this version of Visual Prolog is creation of Linux console applications, for example, CGI applications.
You can develop, debug and test such applications using powerful features of Visual Development Environment (VDE) under Windows.

While creating such projects in the VDE's Application Expert, you should use the following settings: 

"Windows32" for Target Platform, 
"Textmode" for Target UI Strategy., 
In "Other Options" you should turn OFF all packages.

Then you will be able to rebuild the debugged application under UNIX by using supplied LINUX  command line compiler to create a UNIX executable.

Red Hat 6.0 Support

Red Hat 6.0 version of Linux is now supported. 
In Visual Prolog 5.1 it was impossible to link statically with Red Hat 6.0.

Fixed Bugs

Serious bugs found in the  Linux version of Visual Prolog 5.1 have been fixed.
